- **Menu section title** (line 230): restore static "منوی کامل" (remove dynamic x-text)
- **Menu section subtitle** (line 232): restore static text
- **Empty state** (lines 296-321): restore original single empty state for search-not-found only
- **Footer quick links**: remove "علاقه‌مندی‌ها" link (it's now a panel, not a page)
- **Add new panel HTML** before `</body>`: a fixed-position right panel with:
  - Backdrop overlay (`x-show="showFavPanel"`, `@click="showFavPanel=false"`)
  - Panel container (slides in from right, max-width 420px)

## 3. Feedback / Suggestions Section

### Location
New section between About section and Footer in `index.html`.

### HTML structure (`index.html`)
New `<section id="feedback">` with:
- Section title: "انتقادات و پیشنهادات" with gold underline
- Subtitle: "نظر شما برای ما ارزشمند است"
- Form card (centered, max-w-2xl):
  - Name input (optional, placeholder: "نام شما (اختیاری)")
  - Message textarea (required, placeholder: "پیام خود را بنویسید...")
  - Submit button (gold, full-width)
  - Success message after submit (Alpine x-show)
  - Error message if submission fails

### JS changes (`js/app.js`)
- Add state: `feedbackForm: { name: '', message: '' }`, `feedbackSending: false`, `feedbackSuccess: false`, `feedbackError: ''`
- Add method `submitFeedback()`:
  - Validate: message must not be empty
  - Set `feedbackSending = true`
  - Call `SupaDB.submitFeedback({ name: feedbackForm.name || 'ناشناس', message: feedbackForm.message })`
  - On success: set `feedbackSuccess = true`, reset form, after 3s hide success message
  - On error: set `feedbackError = 'خطا در ارسال. لطفاً دوباره تلاش کنید.'`
  - Finally: `feedbackSending = false`

### Supabase client changes (`js/supabase-client.js`)
- Add `submitFeedback(feedback)` method:
  - If `!this.ready`: save to localStorage array `cafe_feedbacks` as fallback
  - If ready: insert into `feedbacks` table with `{ name, message, created_at: new Date().toISOString() }`
  - Fire-and-forget call to `cleanOldFeedbacks()` after successful insert
- Add `cleanOldFeedbacks()` method:
  - If `!this.ready`: return (localStorage doesn't need cleanup)
  - Delete rows where `created_at < 7 days ago` from `feedbacks` table

### SQL — New `feedbacks` table + auto-delete (`seed.sql`)
Append to `seed.sql`:
```sql
-- Feedbacks table
CREATE TABLE IF NOT EXISTS public.feedbacks (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT DEFAULT 'ناشناس',
  message TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_feedbacks_created_at ON public.feedbacks (created_at);

ALTER TABLE public.feedbacks ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anonymous insert" ON public.feedbacks FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow authenticated select" ON public.feedbacks FOR SELECT USING (auth.role() = 'authenticated');
CREATE POLICY "Allow authenticated delete" ON public.feedbacks FOR DELETE USING (auth.role() = 'authenticated');

-- Auto-delete function (call from client or pg_cron)
CREATE OR REPLACE FUNCTION delete_old_feedbacks()
RETURNS void AS $$
BEGIN
  DELETE FROM public.feedbacks WHERE created_at < now() - INTERVAL '7 days';
END;
$$ LANGUAGE plpgsql;
```

### CSS changes (`css/main.css`)
- `.feedback-section`: gradient background matching about section
- `.feedback-form`: card styling (rounded-2xl, card-bg, border, padding)
- `.feedback-input` / `.feedback-textarea`: styled inputs matching site aesthetic
- Success/error message styling
- All dark mode compatible via CSS variables
